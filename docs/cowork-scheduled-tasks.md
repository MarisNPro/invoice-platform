# Cowork Scheduled Task Templates
## Invoice Platform — copy these into Cowork's scheduled tasks

These five templates automate the most common invoice management workflows.
Paste each prompt into **Cowork → Settings → Scheduled Tasks → Add Task**.

**Prerequisites:**
- Invoice Platform MCP server running (`pnpm dev` in `apps/mcp-server/`)
- Cowork connected to the MCP server at `localhost:4020`
- Local folder structure created: `InvoicePlatform/{inbox,processed,reports,outbox}/`

> Folder paths use forward slashes. On Windows replace with backslashes
> or use `~/InvoicePlatform/` which works on all platforms via Cowork.

---

### 1. Nightly inbox processor
**Schedule:** every day at 00:00

**Prompt:**
```
Check the InvoicePlatform/inbox/ folder on my computer.
For each PDF file found:
1. Upload it to the invoice platform using the upload_import MCP tool
2. Extract invoice data using extract_import
3. If confidence >= 0.85, confirm automatically using confirm_import
4. If confidence < 0.85, leave as REVIEWING for me to check tomorrow morning
5. Move the processed file to InvoicePlatform/processed/ with today's date
   prefix: YYYY-MM-DD-originalname.pdf
6. Save a summary of what was processed to
   InvoicePlatform/reports/inbox-YYYY-MM-DD.txt

In the summary include: files processed, invoices auto-confirmed,
files left for manual review, and any errors encountered.
If the inbox is empty, write "Inbox empty — no files to process." to the report.
```

### 2. Weekly overdue report
**Schedule:** every Monday at 08:00

**Prompt:**
```
Get all overdue invoices from the invoice platform.
Create a report file at InvoicePlatform/reports/overdue-YYYY-MM-DD.md with:

- Total overdue amount in EUR
- List of invoices grouped by:
    1–14 days overdue
    15–30 days overdue
    31+ days overdue
- For each invoice: number, customer name, amount, due date, days overdue
- Recommended action per group:
    1–14 days → send polite reminder email
    15–30 days → send firm reminder, flag for follow-up call
    31+  days → escalate, consider generating a dunning message

At the top of the report add a one-line executive summary:
"As of YYYY-MM-DD: N invoices overdue, total EUR X,XXX.XX"
```

### 3. Monthly VAT report
**Schedule:** 1st of every month at 06:00

**Prompt:**
```
Generate the monthly VAT report for last month (the calendar month that
just ended). Use the invoice platform to fetch all invoices with status
SENT or PAID issued during that month.

Create a report file at InvoicePlatform/reports/vat-YYYY-MM.md with:

SUMMARY
- Reporting period: 1 MONTH YYYY – last day of MONTH YYYY
- Total invoiced (net):   EUR X,XXX.XX
- Total VAT collected:    EUR X,XXX.XX
- Total invoiced (gross): EUR X,XXX.XX

VAT BREAKDOWN BY RATE
For each VAT rate used (e.g. 0%, 9%, 21%, 22%, 25.5%):
- Rate: XX%
- EN 16931 category code: S / Z / E / AE
- Taxable amount: EUR X,XXX.XX
- VAT amount:     EUR X,XXX.XX
- Invoice count:  N

INVOICE LIST
A table of all included invoices:
Number | Customer | Issue Date | Net | VAT % | VAT | Gross

Footer note: "This report is for reference only. Verify with your
accountant before filing. Generated by Invoice Platform on YYYY-MM-DD."
```

### 4. Monthly recurring invoices
**Schedule:** 1st of every month at 07:00

**Prompt:**
```
Check the file InvoicePlatform/recurring/templates.json on my computer.
This file contains a list of recurring invoice templates in this format:
[
  {
    "customerName": "Acme Oy",
    "description": "Monthly retainer — software development",
    "amount": 3200,
    "vatRatePercent": 21,
    "unitCode": "MON",
    "currency": "EUR",
    "dueDays": 30,
    "active": true
  }
]

For each template where "active" is true:
1. Create a new invoice via the invoice platform MCP tools
2. Set issue date to today (1st of the month)
3. Set due date to today + dueDays
4. Use the customerName to find the matching contact
5. Save the created invoice PDF to InvoicePlatform/outbox/ using save_to_local

After processing all templates, write a summary to
InvoicePlatform/reports/recurring-YYYY-MM.txt listing:
- Invoices created (number, customer, amount)
- Any templates skipped (customer not found, etc.)
- Total monthly recurring revenue: EUR X,XXX.XX

If templates.json does not exist, create it with an empty array []
and write a note: "No recurring templates configured. Edit
InvoicePlatform/recurring/templates.json to add your recurring clients."
```

### 5. Daily summary
**Schedule:** every day at 08:00

**Prompt:**
```
Fetch a summary of my invoice platform activity and write it to
InvoicePlatform/reports/daily-YYYY-MM-DD.md.

Include these sections:

TODAY'S ACTION REQUIRED
- Invoices due today (number, customer, amount)
- Imports awaiting review (filename, confidence %, created time)
- Draft invoices older than 7 days (number, customer, created date)

THIS WEEK SO FAR (Mon–today)
- Invoices sent: N (total EUR X,XXX.XX)
- Invoices paid: N (total EUR X,XXX.XX)
- New imports processed: N auto-confirmed, N awaiting review

OVERDUE SNAPSHOT
- Total overdue: N invoices, EUR X,XXX.XX
- Oldest overdue: invoice number, customer, N days

CASH FLOW (next 14 days)
- Expected from outstanding invoices: EUR X,XXX.XX
- Invoices due this week: list with amounts

End with a one-line status:
"✓ All clear" if nothing requires action today, or
"⚠ N items need your attention" if there are action items.

Keep the report concise — it should be readable in under 60 seconds.
```
